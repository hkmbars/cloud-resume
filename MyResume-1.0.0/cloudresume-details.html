Building My Cloud Resume Challenge: A Hands-On Learning Journey

How It Started

After graduating from the AWS re/Start program and earning my AWS Certified Cloud Practitioner certification, I started applying for jobs. During one interview, I was asked, "What cloud projects have you done?" I froze. Up until that point, I had felt confident in my answers, but I hadn’t anticipated this question.

I had been practicing cloud exercises, but I wasn’t sure if they counted as actual projects. After the interview, I researched cloud projects that could help me develop my skills further. The Cloud Resume Challenge was the most recommended, so I decided to take it on.

Navigating the Challenge

To complete the challenge, I relied on numerous blog posts, YouTube videos, countless Google searches, and ChatGPT prompts. I even took the time to study the basics of Python and Terraform. At times, it felt overwhelming. Despite following clear instructions, I still encountered challenges—but that’s where the real learning happened.

Troubleshooting issues and finding solutions deepened my understanding of cloud technologies, reinforcing that hands-on experience is the best way to grow.

Note: This blog provides an overview of the key steps I took rather than a detailed, step-by-step guide. I also created a separate personal documentation with an in-depth process, including explanations of each configuration, for my own reference and learning.

Frontend

Hosting a Static Website with a Custom Domain

Website Development

I started by looking up HTML5 templates online and customizing one to fit my content. Initially, I created a draft and later refined it after completing the Cloud Resume Challenge, ensuring that the final version aligned with my technical implementation.

Configuring S3 and CloudFront

To host my static website, I:

  • Created an S3 bucket to store my website files
  • Blocked public access for security
  • Disabled static website hosting since CloudFront would handle content delivery

Instead of serving the website directly from S3, I used CloudFront, which fetches objects from the bucket. This approach improves security and performance.

Setting Up a Custom Domain

I registered a domain and configured the necessary DNS records. Instead of using Route 53, I opted for Hostinger since it offered a better deal for hkmmb.cloud.

After setting up a CloudFront distribution, I:

  • Assigned a custom domain
  • Requested an SSL certificate via AWS Certificate Manager (ACM)
  • Manually updated the DNS records in Hostinger to complete domain validation

This step was especially valuable, as it enhanced my understanding of DNS records and validation processes. Once the certificate was issued, I linked it to CloudFront to enable HTTPS, ensuring secure access to my static website.

Backend

Implementing a Page View Counter

To track website visits, I needed a database and an API to fetch visitor counts.

Database & API Setup

I first set up a DynamoDB table to store the view count data, then created a Lambda function to interact with the database.

Rather than fetching data directly from DynamoDB, I:

  • Developed a Lambda function to handle data retrieval
  • Used Lambda Function URLs to expose it as an API with a dedicated HTTP(S) endpoint
  • Enabled CORS (Cross-Origin Resource Sharing) to restrict access to my website only

Retrieve and Display the View Count

  • Wrote Python code in the Lambda function to query DynamoDB
  • Created a JavaScript code to call the Lambda API and retrieve the visitor count
  • Integrated the JavaScript code into my HTML to dynamically update the website

This setup ensures that my frontend does not interact directly with the database, improving security and scalability.

Automating Website Deployment with GitHub Actions

To streamline deployment, I created a GitHub repository for my website files and set up a CI/CD pipeline using GitHub Actions.

How the Workflow Works
  • Trigger: The workflow runs whenever a push is made to the main branch
  • Fetches Code: Uses actions/checkout to retrieve the latest repository changes
  • Syncs with S3: Uses jakejarvis/s3-sync-action to update the website directory in the S3 bucket
Ensuring Secure Deployments

Rather than hard-coding AWS credentials in my YAML file, I used GitHub Secrets to store:

  • S3 Bucket Name
  • AWS Access Key ID
  • AWS Secret Access Key

With this setup, each code update automatically deploys to my S3 bucket, ensuring the latest version is updated without manual intervention.

Next Steps & Reflections

Although I successfully completed the challenge, I see opportunities for further improvements. One enhancement I plan to implement is integrating AWS Lambda with SES to add email functionality. Additionally, I want to deepen my studies for the AWS Solutions Architect – Associate (SAA) certification.

This challenge reinforced that there’s no single way to solve a problem, there are multiple approaches, each with trade-offs. Completing this project gave me a strong sense of accomplishment, highlighting how much I’ve learned and how much more there is to explore.

I hope this inspires others to take on the Cloud Resume Challenge and embark on their own cloud learning journey. Thank you for taking the time to read!
